下面介绍一下案例Demo的API调用方式：

请求方式 URL 说明 测试（注意打开浏览器console查看）
POST /api/demo 添加（需提供需要添加的demo信息） 添加demo(name)
GET /api/demo/1 查看id为1的demo记录 获取id为1的demo记录
DELETE /api/demo/1 删除id为1的demo记录 删除id为1的demo记录
PUT /api/demo/1 更新id为1的demo记录（需提供需要更新的demo信息） 更新id为1的demo记录为demo(name-edit)

注意：fo-ssm的请求方式使用restful风格，不懂的可以搜索restful风格了解释义。
